# AI Interaction Guidelines

Этот файл описывает правила взаимодействия AI-ассистента с автором проекта (@pooow) в рамках репозитория `cpro-autotest`.

## Стиль работы
1.  **Маленькие шаги**: Двигаемся итеративно. Не пишем 5 файлов за раз. Один шаг — одна логическая задача.
2.  **Полные файлы**: При изменении кода предоставлять **полную** новую версию файла для копирования (Full File Replacement) в виде вложения, а не diff.
3.  **TDD (Test Driven Development)**: Сначала пишем тест (или обсуждаем проверку), потом код.
4.  **Git Commits**:
    - Всегда предлагать сделать коммит после успешного шага.
    - Предлагать готовое название коммита (Conventional Commits: `feat:`, `fix:`, `refactor:`).
    - Напоминать про `git commit --amend`, если исправляем ошибку в рамках текущей задачи.

## Требования к коду
1.  **Комментарии**: Код должен быть щедро прокомментирован (Docstrings, inline-комментарии) на русском языке. Объяснять "почему", а не только "что".
2.  **Логирование**:
    - Использовать `logging` и `coloredlogs`.
    - Вывод в консоль должен быть подробным, цветным и понятным (структурированным).
    - Не спамить ошибками (traceback) в ожидаемых ситуациях (например, ожидание IP).
3.  **Конфигурация**: Приоритет параметров: CLI аргументы > Config (YAML) > Hardcode defaults.

## Контекст
- Автор работает на **ALT Linux** (nvchad, zsh, lazygit).
- Инфраструктура: **Proxmox VE** (два сервера: `r` и `pve9`), разные ключи и хранилища.
- Репозиторий на GitHub: `pooow/cpro-autotest`.
- Всегда отталкиваться от последнего коммита в ветке `main`.
